## Overview

CephaloAI is a single-page application that lets clinicians upload a lateral cephalogram, automatically detect anatomical landmarks, compute 8 standard cephalometric angles, and receive an AI-generated orthodontic diagnosis with clinical recommendations — all in one workflow.

---

## Features

- **Lateral X-ray upload** — drag-and-drop or file picker (JPG, PNG, WEBP)
- **Automated landmark detection** — overlay image highlighting detected points
- **8 cephalometric measurements** — each classified as Normal / Poor / Critical against standard ranges
- **AI diagnosis summary** — free-text narrative generated by the diagnostic API
- **Clinical recommendations** — actionable list returned alongside the diagnosis
- **Responsive layout** — single-column on mobile, 3-column grid on XL screens
- **Error boundary** — graceful recovery from render errors without full-page crashes

## Architecture

### Data Flow

```
┌─────────────┐     multipart/form-data      ┌──────────────────────┐
│ ImageUpload │ ─────────────────────────── ▶ │ POST /process-image  │
│  (file)     │                               │                      │
└─────────────┘                               └──────────┬───────────┘
                                                         │
                                            { image_with_overlay_path,
                                              angles: { ANB, APDI, … } }
                                                         │
                                                         ▼
                                              ┌──────────────────────┐
                                              │ POST /diagnosis      │
                                              │ (JSON angles)        │
                                              └──────────┬───────────┘
                                                         │
                                            { diagnosis: string,
                                              recommendations: string[] }
                                                         │
                                                         ▼
                                              ┌──────────────────────┐
                                              │  DiagnosticPanel     │
                                              │  (table + summary +  │
                                              │   recommendations)   │
                                              └──────────────────────┘
```

**State management:** all state lives in `App.tsx` via `useState` — no external store.

### Component Responsibilities

| Component | Responsibility |
|---|---|
| `App.tsx` | Application state, API calls, layout orchestration |
| `ImageUpload` | File input with drag-and-drop, preview trigger |
| `ImageDisplay` | Side-by-side original / analyzed image panels |
| `DiagnosticPanel` | Measurements table, diagnosis summary, recommendations list |
| `ErrorBoundary` | Catches render errors, renders fallback UI |
| `src/components/ui/` | Shared primitives (Button, Card, Badge, Separator, ScrollArea) built on Radix UI + Tailwind |
| `src/constants/diagnostics.ts` | Normal ranges per measurement, status color map |
| `src/types/angles.ts` | `Angles` and `AngleValue` TypeScript interfaces |

## Cephalometric Measurements

| Measurement | Normal Range | Description |
|---|---|---|
| **ANB** | 2° – 4° | Sagittal jaw relationship (A-point to Nasion to B-point) |
| **APDI** | 81° – 87° | Anteroposterior dysplasia indicator |
| **FHI** | 0.65 – 0.75 | Facial height index (ratio of anterior/posterior face height) |
| **FMA** | 22° – 28° | Frankfort-Mandibular plane angle; indicates vertical growth pattern |
| **MW** | 25 – 30 mm | Maxillary width measurement |
| **ODI** | 72° – 88° | Overbite depth indicator |
| **SNA** | 82° ± 2° | Sagittal position of the maxilla relative to cranial base |
| **SNB** | 80° ± 2° | Sagittal position of the mandible relative to cranial base |

## API Reference

### `POST /process-image`

Upload a lateral cephalogram for landmark detection and angle computation.

**Request:** `multipart/form-data`
```
file: <image file>
```

**Response:**
```json
{
  "image_with_overlay_path": "/tmp/output/overlay_abc123.png",
  "angles": {
    "ANB":  { "value": 3.2,  "class": "1" },
    "APDI": { "value": 84.1, "class": "1" },
    "FHI":  { "value": 0.71, "class": "1" },
    "FMA":  { "value": 25.0, "class": "1" },
    "MW":   { "value": 27.5, "class": "1" },
    "ODI":  { "value": 80.3, "class": "1" },
    "SNA":  { "value": 82.5, "class": "1" },
    "SNB":  { "value": 79.3, "class": "2" }
  }
}
```

---

### `GET /download-image/{filename}`

Download the processed overlay image. The filename is extracted from `image_with_overlay_path` in the `/process-image` response.

---

### `POST /diagnosis`

Generate an AI diagnosis from the computed angle measurements.

**Request:** `application/json`
```json
{
  "ANB":  { "value": 3.2,  "class": "1" },
  "APDI": { "value": 84.1, "class": "1" },
  "FHI":  { "value": 0.71, "class": "1" },
  "FMA":  { "value": 25.0, "class": "1" },
  "MW":   { "value": 27.5, "class": "1" },
  "ODI":  { "value": 80.3, "class": "1" },
  "SNA":  { "value": 82.5, "class": "1" },
  "SNB":  { "value": 79.3, "class": "2" }
}
```

**Response:**
```json
{
  "diagnosis": "The patient presents a Class I skeletal relationship with normal facial proportions...",
  "recommendations": [
    "Schedule follow-up in 6 months to monitor growth",
    "Evaluate occlusal relationships clinically"
  ]
}
```

---

### Status Classes

| Class | Label | Meaning |
|---|---|---|
| `"1"` | Normal | Within the standard reference range |
| `"2"` | Poor | Borderline — monitor or investigate |
| `"3"` | Critical | Outside acceptable range — clinical action warranted |

## Getting Started

### Prerequisites

- Node.js 18 or later
- npm 9 or later
- Running instances of both backend services (see [Environment Variables](#environment-variables))

### Install

```bash
git clone <repository-url>
cd frontent
npm install
```

### Configure

```bash
cp .env.example .env
```

Edit `.env` and set both backend URLs:

```env
VITE_API_URL=http://localhost:5000   # Image processing API
VITE_DIAG_URL=http://localhost:3001  # Diagnostic AI API
```

### Run

```bash
npm run dev
```

The app is served at `http://localhost:5173` by default.
